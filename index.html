About me

Hi, my name is Hassan Kamara and I am current student at the University of Rochester studying Economics and Psychology. On my free time, I love to work on technical projects involving a myriad of technologies, including but not limited to C/C++, Ruby (on Rails), Javascript, HTML5/CSS3, and PHP.

One of my most recent projects involved creating a word search game in C++. I used a word bank inside the source code to hold a predetermined set of words. I then used if/else statements nested inside a while loop to cross-reference the words I queried from the user with the word-bank in my source to see if there's a match. I also implemented a scalable timer to run alongside the main code so that the user can try to get all 18 words in 60 seconds.

I am currently working with Ruby on Rails and databases; the end goal is to connect client-side and server side technologies to create rich, multi-page web applications.

Bass Music

Music to our ears.

This was a challenging but ultimately rewarding project because it was the first time I worked with the has_many / belongs_to association in the Ruby on Rails framework. An album has_many songs and songs belong_to an album. With this in mind, I needed to create two pieces of data: album and track model. An album needs a cover, title, and artist and a track needs the song's name and length.

I then set up an Album controller to handle requests. I created a method, index, that retrieved all albums from the database and stored them in the variable @albums. Next I created a second method, show, that retrieves a single album from the database, saves it in @albums, and then does the same for tracks, saving them in @tracks. Finally, I looped through the tracks and albums to display the information

C++ Word Search

Can you find them all?

My first idea was to design a search algorithm that searches the rows and columns of a 2D array for words the user specifies. But I ran into difficulties so I couldn't go that route. Then I tried to use a switch statement but I ran into problems there too. Switch statements, to the best of my knowledge, only accept 'char', not 'strings', so I couldn't do that either. So for instance, I can't use "spade"--one of the words on the puzzle. I finally decided to use a wordbank (think of it as a database), hidden from the user, and used if-else statements to check for words the user wants.

From there, I wrapped the if-else statements in a while loop, to continuously ask the user for new words (while totaling up the points they accumulate). When the user has earned 18 points (so the while loop ran while points is less than 18), they receive a congratulatory message. Also, because I used an if-else statement, input validation was simple -- any input that (a) wasn't a string or (b) a number is automatically flagged and rejected. User is asked to enter a valid input.

Threadly

Penny for your thoughts?

This was one of my favourite projects because it was the first time I worked with a database. I used the request/response cycle:

First Turn

User [Request] ⇢ Browser ⇢ Rails Router ⇢ Controller ⇄ View ⇢ Controller ⇢ Browser ⇢ User

Second Turn

User [Request] ⇢ Browser ⇢ Rails Router ⇢ Controller ⇄ Model ⇄ Database ⇄ Model ⇢ Controller ⇄ View ⇢ Controller ⇢ Browser ⇢ User

as a framework to fetch and store user posts. From there, I used an index controller to handle GET requests and a POST controller to handle post requests.
